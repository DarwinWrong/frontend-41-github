## 1. Створення та переміщення до папки з проєктом

```bash
# створення папки Frontend
md Frontend
# перехід в папку Frontend
cd Frontend

md "Frontend 41"
cd "Frontend 41"

md "D:\IDE\Frontend 41"
cd "D:\IDE\Frontend 41"
```

- Переходить до папки, в якій зберігається проєкт.

```bash
ls -Force
```

- Перегляд файлів windows

```bash
ls -a
```

- Перегляд файлів mac, linux

## 2. Клонування репозиторію

```bash
git clone https://github.com/DarwinWrong/frontend-41-github.git frontend-41-git
```

- Клонує віддалений репозиторій у нову папку `frontend-41-git`.
- Створить локальний репозиторій з усіма файлами і комітами з віддаленого репозиторію.
- Автоматично встановить віддалений сервер під назвою origin, що вказує на вихідний репозиторій.

```bash
git remote rename origin my-remote
```

- Після цього ваш віддалений репозиторій буде називатися my-remote.

## 3. Оновлення репозиторію

```bash
git fetch origin гілка_ім'я
```

- Завантажує зміни з віддаленого репозиторію, але не застосовує їх до локальної гілки

```bash
git fetch --all
```

- Завантажує всі зміни з віддаленого репозиторію без злиття з поточною гілкою.

```bash
git pull
```

#оновити усі гілки.

```bash
git pull гілка_ім'я
```

#оновити локальну гілку.

```bash
git pull origin гілка_ім'я
```

#оновити віддалену гілку.

- Оновлювати свої гілки перед кожними новими змінами. Виконує git fetch, а потім об’єднує зміни з локальною гілкою.

## 4. Ініціалізація локального репозиторію

```bash
git init
```

- Ініціалізує новий локальний Git-репозиторій у поточній папці. Викликати у випадку якщо ви не клонували віддалений репозиторій.

## 5. Налаштування користувача Git

```bash
git config user.name "IT Step Academy"
git config user.email "user@itstep.academy"
```

- Задає ім'я користувача та електронну пошту для комітів у цьому репозиторії.

## 6. Ігнорування файлів ,gitignore

### Створення .gitignore

```bash
touch .gitignore
```

```bash
New-Item -ItemType File .gitignore
```

- Створює файл .gitignore у кореневому каталозі вашого репозиторію. Linux,Mac -> touch, Windows -> New-Item.

<ol>
<h4>Чого не повинно бути в репозиторії. Ігнорування файлів</h4>
<li>файли проміжних та виконавчих кодів (.exe, .dll, .class, .jar, .obj тощо);</li>
<li>файли локальних налаштувань IDE (.vs, .idea, тощо);</li>
<li>файли додаткових бібліотек та модулів, встановлених з зовнішніх джерел (власні бібліотеки та модулі, очевидно, можуть бути в репозиторії);</li>
<li>системні (зазвичай сховані) файли та каталоги (Thumbs.db, __MACOSX, .DS_Store, .lock, .tmp тощо);</li>
<li>файли звітів та журналів логування (.log, .err);</li>
<li>не обов'язково, але дуже бажано вилучати з репозиторію файли з паролями до сторонніх ресурсів (баз даних, облікових записів, платіжних систем тощо).</li>
</ol>

### Приклад встановлення правил для ігнорування відстеження файлів в .gitignore

```bash
# назва одного файлу;
file.exe
# назва каталогу;
bin/
# файли з довільним іменем і з розширенням «.exe»;
*.exe
# усі файли та каталоги, що знаходяться у каталозі «obj»
obj/**
# цей файл не ігнорувати.
!/plugins/lzb-alkima-select-entities
```

- Приклад способів добавлення файлів в .gitignore.

### Очистіть кеш Git

```bash
git rm -r --cached .
```

- Очищення кешу для файлів, які вже відстежуються.

### Застосування змін

```bash
git add .gitignore
git commit -m "Add .gitignore file"
```

- Додайте .gitignore та застосуйте зміни.

## 7. Перегляд статусу та історії комітів

### Перевірка статусу файлів

```bash
git status
```

- Показує поточний статус файлів у робочій директорії та індексі.

### Перегляд історії комітів

```bash
git log
```

- Відображає історію комітів у поточній гілці.

### Більш читабельний перегляд історії комітів активної гілки

```bash
git log --oneline --graph --decorate
```

- Відображає історію комітів активної гілки з графом.

### Більш читабельний перегляд історії комітів усіх гілок(локальні+віддалені)

```bash
git log --oneline --graph --decorate --all
```

- Відображає історію усіх комітів усіх гілок з графом.

### Відображення деталей конкретного коміту

```bash
git show 9688247
```

- Показує зміни та метадані коміту з ID `9688247`.

## 8. Перелік відстежуваних файлів

```bash
git ls-files
```

- Виводить список усіх файлів, які відстежує Git у поточному репозиторії.

## 9. Робота з гілками

### Перегляд локальні гілки:

```bash
git branch
```

### Перегляд усі гілки:

```bash
git branch -a
```

### Створення нової гілки та перехід до неї

```bash
git checkout -b ivanenko
```

```bash
git switch -c ivanenko
```

- Створює нову гілку `ivanenko` і перемикається на неї.

### Створення нової гілки з коміту

```bash
git checkout -b <new-branch-name> <commit-hash>
```

```bash
git switch -c <new-branch-name> <commit-hash>
```

- Створює нову гілку з коміту.

### Перехід до існуючої гілки

```bash
git checkout ivanenko
```

```bash
git switch ivanenko
```

- Перемикається на існуючу гілку `ivanenko`.

### Перейменування гілки

```bash
git branch -m стара_назва нова_назва
```

- Перейменовує гілку з `стара_назва` на `нова_назва`.

### Видалення гілки на локальному репозиторії

```bash
git branch -d назва_фічі
```

- Видаляє гілку `назва_фічі` з локального репозиторію.

### Видалення гілки на віддаленому репозиторії

```bash
git push origin --delete стара_назва
git push origin -d стара_назва
```

- Видаляє гілку `стара_назва` з віддаленого репозиторію.

### Відновлення видаленої гілки

```bash
git reflog
```

- Відобразить список усіх дій, включаючи видалення та створення гілок.
- Знайдіть останній коміт вашої видаленої гілки.

```bash
git checkout -b <назва_гілки> <хеш_коміту>
git checkout -b deleted-branch f29e4d2
```

- Створить гілку заново з цього коміту

### Встановлення відстеження для нової гілки

```bash
git push --set-upstream origin нова_назва
```

- Запускає гілку `нова_назва` на віддаленому репозиторії та налаштовує відстеження.

## 10. Додавання файлів до коміту

### Додавання окремого файлу

```bash
git add filename
```

- Додає конкретний файл до коміту.

### Додавання декілька файлів

```bash
git add filename1 filename2
```

- Додає декілька файлів до коміту.

### Додавання всіх змінених файлів

```bash
git add .
```

- Додає всі змінені файли до коміту.

## 11. Створення коміту

```bash
git commit -m "Name of commit"
```

- Зберігає всі зміни з повідомленням коміту `"Name of commit"`.

## 11.1 Створення коміту з додаванням усіх змінених файлів

```bash
git commit -a -m "Name of commit"
```

- Додає усі змінені файли до коміту та Зберігає їх з повідомленням коміту `"Name of commit"`.

## 12. Зміна поточного коміту

```bash
git commit --amend -m "New commit message"
```

- Виконайте команду для зміни повідомлення останнього коміту.

  Якщо коміт уже було відправлено на віддалений репозиторій, потрібно форсувати(перезаписати) оновлення, щоб зміни відобразилися:

```bash
git push --force
```

## 13. Скасування коміту

```bash
git revert HEAD
```

- Щоб скасувати останній коміт (внесені зміни). Це створить новий коміт, який зворотний до останнього коміту, і скасує зміни, внесені в цей коміт

```bash
git revert abc123
```

- Скасувати певний коміт (наприклад, з ID abc123), не зачіпаючи наступні коміти.

### Скасування останнього коміту, зберігаючи внесені зміни

```bash
git reset --soft HEAD^
```

- Видаляє останній коміт, але залишає файли як змінені для наступного коміту.

### Повне скасування останнього коміту, не зберігаючи внесені зміни

```bash
git reset --hard HEAD^
```

- Видаляє останній коміт і повертає файли до стану перед комітом.

```bash
git checkout -- index.html
```

- Видаляє всі незбережені зміни у файлі index.html і поверне його до стану з останнього коміту. git checkout перезаписує зміни в робочій директорії: Якщо у файлі є незбережені зміни, вони будуть втрачені.

### Перегляд будь-якого коміту за допомогою commit-hash

```bash
git switch --detach <commit-hash>
git checkout <commit-hash>
```

- Ви можете переглянути конкретний коміт перейшовши на нього, але не вносити зміни в історію гілки. Ви можете створите нову гілку з обраного коміту і вносити зміни.

## 14. Злиття гілок

```bash
git checkout імя_гілки_1
git merge імя_гілки_2
```

- Об'єднує гілку "імя*гілки_2" з гілкою "імя*гілки_1".

## 15. Відправка змін на віддалений репозиторій

```bash
git push origin ivanenko
```

- Відправляє зміни з поточної гілки на віддалений репозиторій у гілку `ivanenko`.

```bash
git push --force origin ivanenko
```

- Відправляє зміни з поточної гілки на віддалений репозиторій у гілку `ivanenko` з перезаписом (перетирання) усіх файлів.

## 16. Видалення файлів з індексу (збереження локально)

```bash
git rm --cached file.tmp
```

- Видаляє файл `file.tmp` з індексу (не видаляючи з локальної файлової системи).

## 17. Відновлення змін

### Відновлення конкретного файлу

```bash
git restore file.tmp
```

- Нова версія команди. Відновлює файл `file.tmp` до попереднього стану.

```bash
git restore --source=develop myfile.txt
```

- Це замінить myfile.txt у вашій робочій директорії на версію з гілки develop.

```bash
git restore --source=develop myfile.txt
```

- Це відновить файл до стану, який він мав у коміті з хешем a1b2c3d.

```bash
git restore --source=main file1.txt file2.txt
```

- Це відновить file1.txt і file2.txt з гілки main.

```bash
git restore --source=HEAD myfile.txt
```

- Це відновить файл до стану, в якому він був після останнього коміту.

```bash
git checkout <branch_or_commit> -- <filename>
```

- Стара версія команди. Замінить файл <filename> на його версію з гілки або коміту <branch_or_commit>, але не перемикатиме вас на саму гілку.

## 18. Як створити Git Alias

<h4>Git Alias — це спосіб скорочення довгих команд, які ви часто використовуєте, у коротші назви.</h4>
<ol>
  <li>
    Введіть команду для створення псевдоніма:
    <code>git config --global alias.<alias_name> "<git_command>"</code>
    <p><alias_name> — назва вашого псевдоніма</p>
    <p><git_command> — оригінальна команда Git</p>
  </li>
    <li>
    Введіть команду для створення псевдоніма:
    <code>git config --global alias.<alias_name> "<git_command>"</code>
    <p><alias_name> — назва вашого псевдоніма</p>
    <p><git_command> — оригінальна команда Git</p>
  </li>
  <li>
    Приклад:
    <code>
      git config --global alias.co checkout
      git config --global alias.br branch
      git config --global alias.cm "commit -m"
      git config --global alias.lg "log --graph --oneline --decorate --all"
      git config --global alias.new "config --global"
      git new alias.cma "commit -a -m"
    </code>
  </li>
  <li>
    Використовуйте псевдонім замість повної команди:
    <code>
      git co main       # Замість git checkout maincm
      git br            # Замість git branch
      git cm "message"  # Замість git commit -m "message"
      git lg            # Виконує git log --graph --oneline --decorate --all
      git new alias.cm "commit -m" # Замість git config --global alias.cm
      git cma "update" # Замість git commit -a -m "update"
    </code>
  </li>
  <li>
    Перевірити всі створені псевдоніми можна так:

```bash
git config --get-regexp alias
```

  </li>
</ol>
