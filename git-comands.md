# Основні Git команди для роботи студентів

Цей документ містить основні команди Git, які допоможуть працювати з репозиторієм, створювати коміти, переглядати зміни та працювати з гілками.

## 1. СТворення та ереміщення до папки з проєктом

```bash
md Frontend
cd Frontend

md "Frontend 41"
cd "Frontend 41"

md "D:\IDE\Frontend 41"
cd "D:\IDE\Frontend 41"
```

- Переходить до папки, в якій зберігається проєкт.

```bash
ls -Force
```

- Перегляд файлів windows

```bash
ls -a
```

- Перегляд файлів mac, linux

## 2. Клонування репозиторію

```bash
git clone https://github.com/DarwinWrong/frontend-41-github.git frontend-41-git
```

- Клонує віддалений репозиторій у нову папку `frontend-41-git`.
- Створить локальний репозиторій з усіма файлами і комітами з віддаленого репозиторію.
- Автоматично встановить віддалений сервер під назвою origin, що вказує на вихідний репозиторій.

## 3. Оновлення локального репозиторію

```bash
git fetch origin гілка_ім'я
```

- Завантажує зміни з віддаленого репозиторію, але не застосовує їх до локальної гілки

```bash
git fetch --all
```

- Завантажує всі зміни з віддаленого репозиторію без злиття з поточною гілкою.

```bash
git pull origin гілка_ім'я
```

- Виконує git fetch, а потім об’єднує зміни з локальною гілкою.

## 4. Ініціалізація локального репозиторію

```bash
git init
```

- Ініціалізує новий локальний Git-репозиторій у поточній папці. Викликати у випадку якщо ви не клонували віддалений репозиторій.

## 5. Налаштування користувача Git

```bash
git config user.name "IT Step Academy"
git config user.email "user@itstep.academy"
```

- Задає ім'я користувача та електронну пошту для комітів у цьому репозиторії.

## 6. Ігнорування файлів ,gitignore

### Створення .gitignore

```bash
touch .gitignore
```

- Створює файл .gitignore у кореневому каталозі вашого репозиторію.

### Добавляємо файли .gitignore

```bash
file.exe — назва одного файлу;
bin/ — назва каталогу;
*.exe — файли з довільним іменем і з розширенням «.exe»;
obj/** — усі файли та каталоги, що знаходяться у каталозі «obj»
!/plugins/lzb-alkima-select-entities – Цей файл не ігнорувати.
```

- Приклад способів добавлення файлів в .gitignore.

### Очистіть кеш Git

```bash
git rm -r --cached .
```

- Очищення кешу для файлів, які вже відстежуються.

### Застосування змін

```bash
git add .gitignore
git commit -m "Add .gitignore file"
```

- Додайте .gitignore та застосуйте зміни.

## 7. Перегляд статусу та історії комітів

### Перевірка статусу файлів

```bash
git status
```

- Показує поточний статус файлів у робочій директорії та індексі.

### Перегляд історії комітів

```bash
git log
```

- Відображає історію комітів у поточній гілці.

### Відображення деталей конкретного коміту

```bash
git show 9688247
```

- Показує зміни та метадані коміту з ID `9688247`.

## 8. Перелік відстежуваних файлів

```bash
git ls-files
```

- Виводить список усіх файлів, які відстежує Git у поточному репозиторії.

## 9. Робота з гілками

### Створення нової гілки та перехід до неї

```bash
git checkout -b ivanenko
```

- Створює нову гілку `ivanenko` і перемикається на неї.

### Перехід до існуючої гілки

```bash
git checkout ivanenko
```

- Перемикається на існуючу гілку `ivanenko`.

### Перейменування гілки

```bash
git branch -m стара_назва нова_назва
```

- Перейменовує гілку з `стара_назва` на `нова_назва`.

### Видалення гілки на локальному репозиторії

```bash
git branch -d назва_фічі
```

- Видаляє гілку `назва_фічі` з локального репозиторію.

### Видалення гілки на віддаленому репозиторії

```bash
git push origin --delete стара_назва
```

- Видаляє гілку `стара_назва` з віддаленого репозиторію.

### Встановлення відстеження для нової гілки

```bash
git push --set-upstream origin нова_назва
```

- Запускає гілку `нова_назва` на віддаленому репозиторії та налаштовує відстеження.

## 10. Додавання файлів до коміту

### Додавання окремого файлу

```bash
git add filename
```

- Додає конкретний файл до коміту.

### Додавання всіх змінених файлів

```bash
git add .
```

- Додає всі змінені файли до коміту.

## 11. Створення коміту

```bash
git commit -m "Name of commit"
```

- Зберігає всі зміни з повідомленням коміту `"Name of commit"`.

## 12. Скасуванн6я коміту

```bash
git revert abc123
```

- Скасувати певний коміт (наприклад, з ID abc123), не зачіпаючи наступні коміти.

### Скасування останнього коміту, зберігаючи зміни

```bash
git reset --soft HEAD^
```

- Видаляє останній коміт, але залишає файли як змінені для наступного коміту.

### Повне скасування останнього коміту

```bash
git reset --hard HEAD^
```

- Видаляє останній коміт і повертає файли до стану перед комітом.

## 13. Відправка змін на віддалений репозиторій

```bash
git push origin ivanenko
```

- Відправляє зміни з поточної гілки на віддалений репозиторій у гілку `ivanenko`.

## 14. Видалення файлів з індексу (збереження локально)

```bash
git rm --cached file.tmp
```

- Видаляє файл `file.tmp` з індексу (не видаляючи з локальної файлової системи).

## 15. Відновлення змін

### Відновлення конкретного файлу

```bash
git restore file.tmp
```

- Відновлює файл `file.tmp` до попереднього стану.
